## 1. Executive Summary

X.IDE Product Page Builder is a comprehensive no-code website builder platform designed for creating high-converting product pages, landing pages, and sales funnels. The platform enables users to build professional pages through a drag-and-drop interface with integrated payment processing, form handling, analytics, and marketing pixel tracking.

### Key Value Propositions

- **No-Code Builder:** Drag-and-drop interface with 16+ pre-built elements
- **Integrated Payments:** LeanX payment gateway with bump offer/upsell support
- **Multi-Channel Publishing:** Subdomain, custom domain, and path-based URLs
- **Analytics & Tracking:** Built-in analytics with Meta Pixel, TikTok, and Google integrations
- **Form Automation:** Lead capture, booking forms, and Google Sheets integration

### 5.6 Row-Level Security Policies

```sql
-- Profiles: Users can only access their own profile
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view own profile"
  ON profiles FOR SELECT
  USING (auth.uid() = id);

CREATE POLICY "Users can update own profile"
  ON profiles FOR UPDATE
  USING (auth.uid() = id);

-- Projects: Users can only access their own projects
ALTER TABLE projects ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can CRUD own projects"
  ON projects FOR ALL
  USING (auth.uid() = user_id);

-- Elements: Users can access elements of their projects
ALTER TABLE elements ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can CRUD elements of own projects"
  ON elements FOR ALL
  USING (
    project_id IN (
      SELECT id FROM projects WHERE user_id = auth.uid()
    )
  );

-- Transactions: Users can view their own transactions
ALTER TABLE transactions ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view own transactions"
  ON transactions FOR SELECT
  USING (auth.uid() = user_id);

-- Published Pages: Public read access
ALTER TABLE published_pages ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Anyone can view published pages"
  ON published_pages FOR SELECT
  USING (true);

CREATE POLICY "Users can manage own published pages"
  ON published_pages FOR ALL
  USING (
    project_id IN (
      SELECT id FROM projects WHERE user_id = auth.uid()
    )
  );
```

## 14. Security

### 14.1 Authentication Security

- **Password Hashing:** bcrypt via Supabase Auth
- **Session Management:** JWT tokens with refresh
- **MFA Support:** TOTP (future)

### 14.2 Data Security

- **Row-Level Security:** All tables protected
- **Input Sanitization:** isomorphic-dompurify
- **SQL Injection:** Parameterized queries via Supabase
- **XSS Prevention:** React automatic escaping + CSP

### 14.3 API Security

- **CSRF Protection:** Token validation on mutations
- **Rate Limiting:** IP-based throttling
- **CORS:** Configured for allowed origins
- **HTTPS:** Enforced in production

### 14.4 Security Headers

```typescript
// next.config.js
headers: [
  {
    key: 'X-Frame-Options',
    value: 'DENY',
  },
  {
    key: 'X-Content-Type-Options',
    value: 'nosniff',
  },
  {
    key: 'X-XSS-Protection',
    value: '1; mode=block',
  },
  {
    key: 'Referrer-Policy',
    value: 'strict-origin-when-cross-origin',
  },
  {
    key: 'Content-Security-Policy',
    value: "default-src 'self'; ...",
  },
];
```

### 14.5 Payment Security

- **PCI Compliance:** LeanX handles card data
- **Webhook Validation:** Signature verification
- **Transaction Integrity:** Idempotency keys
